### 第3页：解决方案总览
- 产品定位：AI 驱动的双向手语翻译系统
- 核心能力：语音↔手语双向翻译、3D虚拟人输出、AI学习陪练
- 差异化：双向翻译 + 3D虚拟人 + 手语文法引擎

- 流程：麦克风 → Web Speech → 语法引擎 → GlossSequence → 3D虚拟人
- 关键：中文→手语语序转换、非手动标记（表情/头势）

### 第5页：功能演示 — 手语转文字

- 流程：摄像头 → MediaPipe关键点 → LSTM分类 → 文字

### 第7页：功能演示 — 手语学习
- 截图：学习页面（词汇查询/跟练/场景演示/数据采集）
- 关键：DTW三维度评分、4个预设场景、数据采集标注工具

### 第8页：技术架构总览
- 架构图：微内核+插件化分层图
- 分层：types → kernel → modules → stores → components → plugins → pages
- 扩展点：插件manifest声明路由/菜单，自动注册

### 第9页：核心技术1 — 手语文法引擎
- 四阶段流水线图：Tokenizer → Rewriter → GlossMapper → NonManualMarker
- CSL重写规则：宾语前移、否定后置、疑问后置、功能词去除
- 非手动标记：疑问句扬眉、否定句摇头、强调句点头

### 第10页：核心技术2 — 3D虚拟人驱动
- 骨骼系统图：18个骨骼节点 + IK求解
- 动作编排：MotionPlayer + TransitionEngine + HandShape插值
- 视觉效果：PBR材质、三点照明、阴影、呼吸动画、雾效

### 第11页：核心技术3 — 手语识别
- 模型架构图：[30,126] → LSTM(128) → LSTM(64) → Dense(64) → Dropout → Dense(softmax)
- 数据增强：镜像/帧丢失/遮挡/Mixup/高斯噪声/时序抖动（6种策略）
- 归一化流程：平移→缩放→时间重采样→展平

### 第12页：核心技术4 — 跟练评分
- 流程图：DTW对齐 → 三维度计算 → 加权评分 → 反馈生成
- 三维度：手形40%（余弦相似度）+ 位置40%（归一化距离）+ 运动20%（角度差）

## 三、演示注意事项

1. **环境准备**：
   - 提前打开浏览器，允许麦克风和摄像头权限
   - 准备好一个安静环境
   - 建议使用 Chrome 浏览器（MediaPipe/TF.js 兼容性最佳）

2. **演示顺序建议**：
   - 先演示「场景演示」模式（最直观，展示双向翻译价值）
   - 再演示「语音转手语」（最惊艳，虚拟人效果）
   - 然后「手语转文字」（技术含量最高）
   - 最后「跟练模式」和「数据采集」（互动性强）

3. **应急方案**：
   - 如果摄像头/麦克风权限失败，使用场景演示模式（不需要权限）
   - 如果 LSTM 模型加载慢，跳过手语识别，重点展示语音转手语
   - 准备一个录屏视频作为备选

4. **关键数据记忆**：
   - 2700万听障人士 / 70万翻译师缺口
   - 首屏 gzip 55KB
   - 548个词汇 / 19个分类
   - 226个单元测试
   - 6种数据增强策略
   - 4个预设演示场景
